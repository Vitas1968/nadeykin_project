# Project instructions

## Language
- Отвечать пользователю на русском языке.
- Код, имена файлов, функций и технические идентификаторы писать на английском.

## Safety
- Не изменять .env, секреты, токены, ключи и пароли.
- Не запускать destructive-команды без явного подтверждения.
- Не удалять файлы без подтверждения.
- Перед изменениями давать краткий план.

## Workflow
- Делать небольшие, проверяемые изменения.
- После изменений запускать доступные проверки.
- Если тестов нет, явно указать, что тесты не запускались.
- Не менять структуру проекта без причины.

## Project goal
Проект предназначен для анализа тендерных документов, извлечения требований, критериев, рисков и подготовки краткой сводки.
